نبذة عن المشروع

القسم 1

🎯 الهدف من المشروع

تم تطوير نظام White Style ليكون منصة متكاملة لإدارة متجر الأزياء الإلكتروني، يجمع بين لوحة تحكم احترافية للإدارة وواجهة عصرية للعملاء، مع التركيز على تجربة المستخدم والأداء العالي.

📋 وصف النظام

نظام White Style هو منصة تجارة إلكترونية متكاملة مصممة خصيصاً لمتاجر الأزياء في فلسطين، يوفر النظام جميع الأدوات اللازمة لإدارة المنتجات، الطلبات، العملاء، والمحتوى بشكل سلس واحترافي.

🎨 واجهة عصرية

تصميم احترافي متجاوب مع جميع الأجهزة (موبايل، تابلت، ديسكتوب) بتقنيات Vue.js 3 الحديثة.

⚡ أداء عالي

بنية تحتية قوية باستخدام Laravel 12 وتقنيات التخزين المؤقت لضمان سرعة استجابة ممتازة.

🔒 أمان محكم

حماية شاملة باستخدام Laravel Sanctum للمصادقة والتحقق من الصلاحيات.

📱 تجربة مستخدم مميزة

سهولة في التصفح والشراء مع نظام سلة مشتريات متطور ونظام مفضلة ذكي.

✨ المميزات الرئيسية

  • لوحة تحكم شاملة لإدارة كافة جوانب المتجر
  • إدارة متقدمة للمنتجات مع دعم الصور المتعددة والألوان والمقاسات
  • نظام طلبات احترافي مع تتبع الحالات والطباعة
  • لوحة إحصائيات تفاعلية مع رسوم بيانية
  • نظام سلة مشتريات ذكي بالتخزين المحلي
  • نظام مفضلة للمنتجات
  • بحث متقدم مع معاينة فورية للنتائج
  • تخصيص ألوان الموقع من لوحة التحكم
  • نظام مدن فلسطينية مع أسعار التوصيل
  • واجهة عربية كاملة بدعم RTL

مكونات النظام

القسم 3

🏗️ معمارية النظام

تم تطوير النظام باستخدام معمارية Frontend/Backend المنفصلة (Decoupled Architecture) لضمان المرونة وسهولة التطوير والصيانة.

📁 المجلد: whiteStyle (Laravel Backend)

يحتوي على الواجهة الخلفية الكاملة وواجهات API:

  • app/Models: 13 موديل (Product, Category, SubCategory, Brand, Size, City, Order, Color, Slider, Faq, Contact, Admin, User)
  • app/Http/Controllers: Controllers منظمة حسب الوظيفة (Admin, Site)
  • database/migrations: هيكل قاعدة البيانات الكامل
  • routes/api.php: جميع مسارات API المحمية والعامة
  • public/storage: تخزين الصور والملفات

💻 المجلد: whiteStyleVue (Vue.js Frontend)

يحتوي على الواجهة الأمامية الكاملة للموقع ولوحة التحكم:

  • src/views/site: صفحات الموقع الإلكتروني للعملاء
  • src/admin: لوحة التحكم الكاملة للإدارة
  • src/stores: Pinia stores لإدارة البيانات
  • src/components: المكونات القابلة لإعادة الاستخدام
  • src/router.js: تعريف جميع المسارات والصفحات

🔄 آلية عمل النظام

Vue.js Frontend (Port 5173) ↓ HTTP Requests (Axios) ↓ Laravel API (Port 8000) ↓ MySQL Database ↓ JSON Response ↓ Vue.js Updates UI

🔗 العلاقات بين الجداول

📊 المنتجات (products):

  • belongsTo Category → category_id
  • belongsTo SubCategory → sub_category_id
  • belongsTo Brand → brand_id
  • sizes → JSON array [1, 2, 3]

📦 الطلبات (orders):

  • belongsTo City → city_id
  • items → JSON array [{product_id, product_name, price, quantity, size}, ...]

📂 الأقسام (categories):

  • hasMany SubCategories
  • hasMany Products

الأمان والحماية

القسم 9

🔒 آليات الأمان المطبقة

1. Laravel Sanctum Authentication

نظام مصادقة قوي باستخدام توكنات Bearer:

  • تسجيل دخول آمن بالبريد وكلمة المرور
  • توليد توكن API فريد لكل جلسة
  • تخزين آمن للتوكنات في قاعدة البيانات
  • إرسال التوكن في header مع كل طلب محمي
  • تسجيل خروج يحذف التوكن من قاعدة البيانات

2. حماية المسارات (Route Protection)

جميع مسارات الإدارة محمية بـ middleware:

Route::middleware('auth:sanctum')->group(function () { // Admin routes here });

3. التحقق من البيانات (Validation)

جميع البيانات الواردة يتم التحقق منها:

  • التحقق من نوع البيانات
  • التحقق من الحقول المطلوبة
  • التحقق من صحة البريد الإلكتروني
  • التحقق من صحة أرقام الهواتف
  • رسائل خطأ واضحة بالعربية

4. حماية CSRF

Laravel يوفر حماية تلقائية من هجمات CSRF

5. تشفير كلمات المرور

جميع كلمات المرور مشفرة باستخدام bcrypt

6. رفع الملفات الآمن

  • التحقق من نوع الملفات المسموحة (images only)
  • تحديد حجم الملف الأقصى
  • تخزين آمن في مجلد storage
  • أسماء ملفات فريدة لتجنب الكتابة فوق الملفات

7. حماية من SQL Injection

استخدام Eloquent ORM و Prepared Statements

8. حماية من XSS

Laravel و Vue.js يقومان بـ escaping تلقائي للنصوص

🛡️ أفضل الممارسات المطبقة

  • فصل Frontend عن Backend لزيادة الأمان
  • استخدام HTTPS في الإنتاج (Production)
  • عدم تخزين معلومات حساسة في LocalStorage
  • رسائل خطأ عامة للمستخدمين (لا تكشف معلومات حساسة)
  • تسجيل (Logging) لجميع عمليات الإدارة الحساسة
  • نسخ احتياطي دوري لقاعدة البيانات
